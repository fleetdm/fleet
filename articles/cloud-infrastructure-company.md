# Cloud infrastructure company automates Linux security workflows with Fleet

A cloud infrastructure company provides hybrid and multi-cloud platforms for enterprise customers. Its internal environment includes Linux systems, macOS devices, and a growing BYOD footprint — all of which need consistent visibility and automated security workflows.

Fleet helps the company automate certificate management and improve visibility across its Linux fleet.

## At a glance

- **Industry:** Cloud computing and infrastructure
- **Devices managed:** Linux pilot fleet with broader environment across operating systems
- **Deployment:** Fleet Cloud (fully managed SaaS)
- **Primary requirements:** Linux visibility, certificate management, GitOps automation
- **Previous challenge:** Limited Linux support and manual security processes

## The challenge: limited Linux support and manual security workflows

Before Fleet, the company struggled with Linux device management. Legacy tools did not provide a strong experience for Linux users, and processes like certificate issuance were manual and difficult to secure. This created risk and added operational overhead for a team that wanted to focus on building infrastructure, not running internal tooling.

The team needed a platform that could automate these workflows and provide better visibility — without taking on additional infrastructure to operate.

## The evaluation criteria

The team prioritized three capabilities:

1. **Linux visibility** — Reliable, real-time data collection across Linux systems to support security and compliance workflows.
2. **Certificate management** — Secure Linux devices using TPM-backed certificates.
3. **GitOps and osquery integration** — Automate workflows through code and collect real-time data across the fleet.

## The solution: managed SaaS with full automation

Fleet gave the team a platform to automate Linux security workflows and improve visibility across the environment.

The company runs on Fleet Cloud, leveraging fully managed SaaS while keeping endpoint data and workflows under the team's control. That choice let the team move quickly without standing up and operating their own Fleet infrastructure — fitting for a company whose engineers already spend their days building cloud platforms for others.

Using Fleet's API, the team syncs user-device relationships, deploys software via GitOps, and automatically triggers compliance checks. This allows the team to maintain a continuous audit state instead of relying on periodic reviews.

Fleet's transparency also helps build trust with technical users who want clear visibility into how their systems are managed.

## The results

Fleet improved both automation and compliance.

- **Stronger Linux security:** Certificate management and policy enforcement are now automated.
- **Continuous compliance:** Real-time checks replace periodic audits.
- **Simpler operations:** Consolidation reduces tool complexity, and Fleet Cloud removes the overhead of running the platform in-house.

## Why they recommend Fleet

For this company, the biggest benefit is the extensibility of Linux management. Fleet provides the flexibility, visibility, and automation needed to manage Linux securely at scale — without the operational burden of running the platform themselves.
